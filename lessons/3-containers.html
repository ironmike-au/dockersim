3.1Container yourself!

Docker containers are instances of an image that you can manipulate. Generally speaking, containers should serve one purpose, like running a web server.

Containers are not full-blown operating systems. They're little copies designed to do one thing well. When a container stops doing that thing it was designed to do, it dies and everyone feels sad for it.

Let's take a look:

docker run ubuntu:14.04 echo "oops"

Skip to Lesson 4 - Docker Fun Files

3.2It's dead, Jim.

R.I.P. Your container lived a short but meaningful life. Mostly.

Seriously. After running echo "oops" your container was terminated and is gone forever.

You can see a history of your containers using the docker ps -a command.

docker ps -a

Skip to Lesson 4 - Docker Fun Files

3.3Interactive Containers

Docker does allow you to run containers interactively, by using the command /bin/bash and attaching a terminal, but this isn't advisable in most cases

Why? Because docker is designed to run containers that live fast and die young.
