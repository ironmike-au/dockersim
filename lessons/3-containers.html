Docker Images

Let's talk about images and containers.

An image is a read-only copy of an operating system. Images live forever, but the only way to change them is to save a new version of that image.

A container is a running (or stopped) instance of an image. Containers die horrible, permanent deaths.

It all starts with images. So let's find a new Ubuntu Server image we can start with.

For easy access, Docker runs the Docker Hub. A repository of many, many images. You can use Hub to download public images, and you can save your own private images there, too. Heck, you can even run your own repository, but let's worry about that later.

Pulling an image from a repository is simple. Type the following command into the terminal window above and see what happens.

sudo docker pull ubuntu:14.04

Uhmm... what just happened? How many things got downloaded just now?

Remember that docker images are read-only. Docker images use layers to record changes from one version of the image to the next. With each of these lines, you see a new version of that image being "layered" on top of the earlier versions.

What you see above are four layers of the Ubuntu 14.04 image. We can see more detail by looking at the image history:

sudo docker history ubuntu:14.04

Here you can see the four layers that make up this image. The oldest (first) update is at the bottom.

Strictly speaking, layers are images on top of each other. That's why each layer has it's own unique image ID, which you can see on the left.
