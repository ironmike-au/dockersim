5.1It's alive!

Do you remember way back in Lesson 3, we talked about how containers lived fast and died young? Even though we've built a fancy little image with Apache on it, if we try to run this image in a new container, it will just exit straight away because we haven't told it to *do* anything.

Because this is webserver, we can run Apache in "FOREGROUND" mode to keep it alive, and thus, keep our container running.

Go ahead and run the command below, and in the next section we'll talk about what this command is actually doing in more detail.

docker run -d -p 80:80 -d littleguy /usr/sbin/apache2ctl -D FOREGROUND

5.2Now I have a container. Ho. Ho. Ho.

Docker has started your container and returned your container's unique ID number. That's all it took!

Because we used the -d flag, Docker ran this container in "detached mode". Without this flag, Docker would have connected us directly to the container, where we might see any output from the apache2ctl process that is running the web server.

With -p 80:80 Docker is

connecting port 80 on our local machine, to port 80 on the container.

Finally, we told Docker the image we wanted to use (littleguy) and the command we wanted to run in that container (apache2ctl).

Want to see your baby? Run the following command to see which containers are currently running.

docker ps
